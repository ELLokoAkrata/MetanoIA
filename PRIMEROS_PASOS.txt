=============================================
PRIMEROS PASOS CON METANOIA
=============================================

¡Bienvenido a MetanoIA! Este es un proyecto educativo de código abierto que te enseñará una nueva forma de programar con inteligencia artificial. A continuación, encontrarás instrucciones paso a paso para comenzar.

---------------------------------------------
REQUISITOS PREVIOS
---------------------------------------------

1. CREAR UNA CUENTA EN GITHUB
   - Visita https://github.com/signup
   - Sigue las instrucciones para crear tu cuenta
   - Verifica tu dirección de correo electrónico

2. CREAR UNA CUENTA EN GROQ
   - Visita https://console.groq.com/signup
   - Regístrate con tu correo electrónico o cuenta de Google
   - Completa el proceso de verificación

3. INSTALAR PYTHON
   - Windows: Descarga el instalador desde https://www.python.org/downloads/ (versión 3.9 o superior)
   - Durante la instalación, marca la casilla "Add Python to PATH"
   - Verifica la instalación abriendo una terminal y escribiendo: python --version

4. INSTALAR GIT
   - Windows: Descarga desde https://git-scm.com/download/win
   - Durante la instalación, selecciona las opciones predeterminadas
   - Verifica la instalación abriendo una terminal y escribiendo: git --version

---------------------------------------------
CONFIGURACIÓN DEL PROYECTO
---------------------------------------------

1. CLONAR EL REPOSITORIO
   - Abre una terminal (CMD o PowerShell en Windows)
   - Navega a la carpeta donde quieres guardar el proyecto
   - Ejecuta el siguiente comando:
     git clone https://github.com/Ellokoakarata/MetanoIA.git
   - Entra al directorio del proyecto:
     cd MetanoIA

2. CREAR UN ENTORNO VIRTUAL (RECOMENDADO)
   - Dentro de la carpeta del proyecto, ejecuta:
     python -m venv venv
   - Activa el entorno virtual:
     * Windows (CMD): venv\Scripts\activate
     * Windows (PowerShell): .\venv\Scripts\Activate.ps1

3. INSTALAR DEPENDENCIAS
   - Con el entorno virtual activado, ejecuta:
     pip install -r requirements.txt

4. OBTENER TU API KEY DE GROQ
   - Inicia sesión en https://console.groq.com/
   - Ve a https://console.groq.com/keys
   - Crea una nueva API key o copia la existente
   - IMPORTANTE: Guarda esta clave en un lugar seguro, no la compartas

5. CONFIGURAR LA API KEY COMO VARIABLE DE ENTORNO
   - Método temporal (solo para la sesión actual):
     * Windows (CMD): set GROQ_API_KEY=tu_api_key_aquí
     * Windows (PowerShell): $env:GROQ_API_KEY="tu_api_key_aquí"
   
   - Método permanente (recomendado):
     * Windows: Busca "Variables de entorno" en el menú de inicio
     * Haz clic en "Variables de entorno..."
     * En "Variables de usuario", haz clic en "Nueva..."
     * Nombre de la variable: GROQ_API_KEY
     * Valor de la variable: tu_api_key_aquí
     * Haz clic en "Aceptar" y cierra todas las ventanas

---------------------------------------------
EJECUTAR LA APLICACIÓN
---------------------------------------------

1. Con el entorno virtual activado y la API key configurada, ejecuta:
   streamlit run app.py

2. Se abrirá automáticamente una pestaña en tu navegador con la aplicación.
   Si no se abre, visita la URL que aparece en la terminal (generalmente http://localhost:8501)

---------------------------------------------
CONSEJOS ADICIONALES
---------------------------------------------

1. EXPLORAR LA DOCUMENTACIÓN
   - Revisa la carpeta docs/ para entender mejor el proyecto
   - Comienza con docs/grimorio-proyecto.md para una visión general

2. CONTRIBUIR AL PROYECTO
   - Si encuentras errores o quieres mejorar algo, puedes crear un "Issue" o un "Pull Request" en GitHub
   - Antes de contribuir, lee docs/process.md para entender el proceso de desarrollo

3. EXPERIMENTAR CON DIFERENTES MODELOS
   - Prueba los diferentes modelos disponibles en la barra lateral
   - Observa cómo cambian las respuestas según el modelo seleccionado

4. PROBLEMAS COMUNES
   - Si ves un error relacionado con la API key, asegúrate de haberla configurado correctamente
   - Si la aplicación no se inicia, verifica que todas las dependencias estén instaladas
   - Si los modelos no cambian correctamente, reinicia la aplicación
